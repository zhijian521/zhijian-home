│   │   ├── posts/              # 公开接口
│   │   ├── admin/              # 后台接口(withAdmin)
│   │   ├── nav/                # 导航站接口
│   │   ├── auth/               # 登录/注册/登出/me

├── components/

│   ├── site/                   # 前台展示组件:文章卡/页头/页脚

## 目录职责

### app/

只放路由、页面、layout、page 私有组件。**业务逻辑不进 app**——调 `lib/domain`。

- `app/*/_components/`:仅该路由树使用的组件,下划线前缀=不参与路由。
- `app/*/_hooks/`:同上,仅页面内 hook。

### components/

| 目录 | 放什么 | 判定标准 |
|---|---|---|
| `ui/` | 原子组件,无业务语义 | `Button` `Dialog` `TextInput`,换站点也能用 |
| `site/` | 前台展示组件 | 文章卡、页头页脚,有站点语义但不绑单一页面 |
| `modules/<域>/` | 业务复合组件 | 把 `ui` 组合成业务区块,如「文章编辑器」「导航书签管理」 |

组件跨 2 个以上页面复用才上移。只用一次的留在页面 `_components`。

### lib/

| 目录 | 放什么 | 依赖方向 |
|---|---|---|
| `core/` | db 连接、auth/session、api-response、http-client、utils | 不依赖 domain |
| `domain/` | posts、taxonomy、nav、uploads、analytics 等业务数据层 | 依赖 core,不依赖 app/components |

**单向依赖**:`app → domain → core`。core 永不反向 import domain。

### showcase / docs

都是「手写 registry」模式,见 [07-文档与校验](./07-文档与校验.md)。

## 文件粒度

- 一个组件一个目录:`foo/foo.tsx` + `foo/foo.module.css`。不用 `foo.tsx` + `foo.module.css` 平铺(除 `ui/` 已有的可保留,新增一律建目录)。
- 一个 domain 一个文件:`domain/posts.ts`,内部聚合该域所有数据访问。
- 不要为「将来可能拆」预留空目录。

## 路径别名

`tsconfig.json` 配 `@/*` 指向 `src/*`。import 一律用 `@/lib/core/db`,不用相对路径跨目录。
