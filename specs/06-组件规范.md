# 06 · 组件规范

## 三层组件

| 层 | 目录 | 判定 | 依赖 |
|---|---|---|---|
| 原子 | `components/ui/` | 无业务语义,换站可用 | 仅 core/utils |
| 前台 | `components/site/` | 前台展示,有站点语义 | ui + domain(只读) |
| 业务复合 | `components/modules/<域>/` | 把 ui 组成业务区块 | ui + domain |

## 文件组织

一个组件一个目录(新增组件一律建目录):

```
components/ui/ghost-button/
├── ghost-button.tsx
└── ghost-button.module.css
```

- 组件文件名 = 导出名 kebab 化。`ghost-button.tsx` → `export function GhostButton`。
- 样式文件同名 `.module.css`,CSS Modules。
- 跨 2 个以上页面复用才上移到 `components/`。只用一次的留 `app/<route>/_components/`。

## 组件写法

- 函数组件,不写 class。
- props 用 interface,显式标注。
- 副作用进 `app/*/_hooks/` 或就近 `useXxx`,组件保持纯展示 + 回调。
- 禁 inline style 写值(除非动态计算值),样式走 module class。

```tsx
interface GhostButtonProps {
  label: string;
  onClick?: () => void;
  variant?: 'default' | 'danger';
}

export function GhostButton({ label, onClick, variant = 'default' }: GhostButtonProps) {
  return (
    <button className={cx(styles.button, styles[variant])} onClick={onClick}>
      {label}
    </button>
  );
}
```

`cx` = clsx(已装依赖),用于条件 class。

## 图标

`components/ui/icons.tsx` 单文件,命名导出每个图标:

```tsx
export function IconSearch(props: SVGProps<SVGSVGElement>) { return ( ... ); }
```

- 不引入图标库——已有的够用,缺了手写一个 SVG。
- 图标可在 `/admin/showcase/icons` 预览(扫描命名导出渲染网格)。

## showcase 登记

`src/showcase/registry.ts` 手写登记,见 [07](./07-文档与校验.md):

```ts
export interface ShowcaseEntry {
  name: string;
  description: string;
  module: 'ui' | 'site' | 'nav' | 'blog' | 'admin';
  source: string;
  examples: { label: string; Component: ComponentType; props?: Record<string, unknown> }[];
}
export const SHOWCASE_REGISTRY: ShowcaseEntry[] = [ /* ... */ ];
```

- 新组件加一行登记 + 示例 props。
- 不做全自动反射(Next 下 fragile,示例不可控)。
- showcase 路由 `/admin/showcase`,复用 admin 鉴权,仅后台可访问。

## 组件去重

发现两个页面写相似组件时:抽到 `components/` 共享,删掉重复。**先看有没有再写**——原项目「复用靠记忆」是反例,v2 用 registry 强制可见。

## 禁止

- 禁组件内写 SQL / 直接调 db——组件走 domain 函数。
- 禁 `ui/` 组件 import `site/` 或 `modules/`(原子不依赖业务)。
- 禁 props 透传超过 2 层(用 context 或上提状态)。
- 禁硬编码样式值(见 [03-样式规范](./03-样式规范.md))。
