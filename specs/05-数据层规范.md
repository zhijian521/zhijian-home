# 05 · 数据层规范

## 分层

```
lib/
├── core/       # 基础设施
│   ├── db.ts              # MySQL 连接池单例
│   ├── auth.ts            # session 读写、requireAdmin/getSession
│   ├── api-response.ts    # success / fail / BizCode / ApiResponse
│   ├── http-client.ts     # 外部 HTTP 调用封装
│   └── utils.ts           # 纯函数工具:日期/字符串/校验
└── domain/     # 业务数据层
    ├── posts.ts           # 文章 CRUD + 查询
    ├── taxonomy.ts        # 分类 + 标签(合并,同域)
    ├── nav.ts             # 导航站:书签/笔记/todos/配置
    ├── uploads.ts         # 文件上传管理
    └── analytics.ts       # 统计采集与查询
```

## 依赖方向

**单向,不可逆**:

```
app  →  domain  →  core
components  →  domain  →  core   (只读 domain 类型/函数)
```

- `core` 永不 import `domain` 或 `app`。
- `domain` 不 import `app`(不碰 Next 的 request/response)。
- `app/api/*/route.ts` 调 `domain` 函数,不直接写 SQL。

## core 边界

### db.ts

- 模块级连接池单例,进程生命周期复用。
- `getDb()` 未配置 `DATABASE_URL` 时返回 `null`,上层决定走示例数据回退。
- 时区 `+08:00`,charset `utf8mb4`,影响 NOW()/DATE() 与中文。
- `getDbConnection()` 取已设时区的独立连接(需要事务时用)。

### auth.ts

- `SessionPayload` 类型定义在此。
- `requireAdminFromRequest(request)` / `getSessionFromRequest(request)` 返回 `SessionPayload | null`。
- 不抛异常——鉴权失败返回 null,由包装器决定响应。
- session 读写细节(jwt/cookie)封装在此,外部不接触。

### api-response.ts

见 [04-API规范](./04-API规范.md)。

## domain 规范

- 一个域一个文件,内部聚合该域所有 SQL + 数据访问函数。
- 函数返回业务对象(类型在文件内定义),不返回 raw `RowDataPacket`。
- 入参做基本校验(id 为正整数、字符串非空),不依赖 route 层。
- 不返回 NextResponse——那是 route 的事。
- 一个函数一个 SQL,不混多个语义。复杂查询用 `?` 占位,禁止字符串拼 SQL(防注入)。

```ts
// domain/posts.ts
export interface Post { id: number; title: string; /* ... */ }

/*== 查全部已发布文章。 ==*/
export async function listPublishedPosts(): Promise<Post[]> {
  const db = getDb();
  if (!db) return SAMPLE_POSTS;
  const [rows] = await db.query<RowDataPacket[]>(
    'SELECT id, title, ... FROM posts WHERE status = ? ORDER BY ...',
    ['published'],
  );
  return rows.map(toPost);
}
```

## 禁止

- 禁 route 内写 SQL——SQL 归 domain。
- 禁 domain import Next 的 `NextRequest`/`NextResponse`。
- 禁 core 反向依赖。
- 禁每个请求新建连接池——单例,见 db.ts。
