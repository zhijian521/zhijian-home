# 00 · 项目总则

## 目标

「知简」是一个个人博客 + 导航站 + 后台管理的 Next.js 站点。v2 从零搭建,目标是:

- **简洁**:能一行写完不写两行;能复用不重写;能不建就不建。
- **整齐**:同类的东西放在同类的地方,命名一致,风格一致。
- **完善**:功能闭环,无半成品;接口有文档,组件有预览,改代码有校验兜底。

## 技术栈

| 项 | 选型 | 备注 |
|---|---|---|
| 框架 | Next.js 15 (App Router) | 不用 Pages Router |
| 语言 | TypeScript 5 | 严格模式,禁止 `any`(信任边界除外,需注释) |
| 运行时 | Node.js | API Route 跑在服务端 |
| 数据库 | MySQL (mysql2/promise) | 连接池单例 |
| 样式 | CSS Modules + 原生变量 | 无 Tailwind / CSS-in-JS |
| Markdown | react-markdown + remark-gfm + rehype-highlight | 前台文章与后台文档 |
| 图表 | recharts | 后台分析 |

不引入以上列表之外的新依赖,除非「stdlib / 已装依赖 / 平台原生」都无法解决。

## 核心原则(YAGNI 优先级)

写任何东西前,按这个顺序自问,停在第一个成立的:

1. **这东西需要存在吗?** 投机性需求直接砍,一句话说明砍了什么。
2. **本项目已有?** 复用现有 helper/类型/组件。先看再写。
3. **stdlib 能做?** 用 stdlib。
4. **平台原生够?** 用原生(`<input type="date">` 优于 picker 库)。
5. **已装依赖能解?** 用已装的。
6. **能一行?** 一行。
7. **以上都不成立**:写最小可用代码。

## 重构边界

- v2 从零重写,不做 re-export 过渡(原项目 REFACTOR.md 的渐进迁移策略不适用)。
- 结构一步到位按理想形态组织,不留「以后再分层」的债。
- 原项目的业务逻辑、SQL、鉴权约定照搬,只重写「结构」与「壳」,不重写「业务」。

## 不做清单(明确排除)

- 不做全自动反射式 showcase / 接口扫描(Next 下 fragile,示例不可控)。
- 不做全局 `modules/` 空壳 CSS 文件。模块差异变量写在模块根组件内。
- 不做 interface-with-one-impl、factory-for-one-product、never-changes-config。
- 不写「为以后预留」的脚手架。以后需要,以后再加。

## 强制校验

每次改动后跑:

```bash
npm run typecheck   # tsc --noEmit
npm run docs:check  # 接口/文档 registry 与磁盘一致性
```

两个都过才算改完。后续接 pre-commit hook 自动化。
