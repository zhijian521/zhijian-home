# 03 · 样式规范

## 三层分层

从上到下覆盖。组件内一律 `var(--xxx)`,不写死值。

| 层 | 文件 | 内容 |
|---|---|---|
| 全局 token | `src/app/tokens.css` | 间距 `--space-*`、字号 `--text-*`、行高、圆角 `--radius` |
| 主题 | `src/app/theme.css` | 颜色、字体族 |
| 模块覆盖 | 模块根组件 CSS Module 内 | 仅该模块差异变量,如 `--nav-bg` |

`globals.css` 顶部顺序:

```css
@import './tokens.css';
@import './theme.css';
/* 其后:全局重置 */
```

tokens 必须在 theme 之前——theme 可以引用 token。

## token 阶梯

- 间距:4px 基准,`--space-1`(4) `--space-2`(8) `--space-3`(12) `--space-4`(16) `--space-6`(24) `--space-8`(32)。
- 字号:rem 单位,`--text-sm` `--text-md` `--text-lg` `--text-xl` `--text-2xl`。
- 圆角:`--radius`(可调,不再全局锁 0)。
- 颜色、字体族只在 `theme.css`,token 层不碰颜色。

## 使用规则

**必须**:`padding: var(--space-3)`、`font-size: var(--text-md)`、`color: var(--text-primary)`。

**禁止**:
- 禁硬编码 `padding: 12px` / `font-size: 14px` / `color: #333`。
- 禁在 `tokens.css` 里放颜色(颜色归 theme)。
- 禁建全局 `modules/` 空壳 CSS。模块差异变量写在模块根组件 CSS Module 的容器选择器内。

## CSS Modules

- 每个组件一个 `.module.css`,与组件同目录同名。
- class 命名 camelCase:`.submitButton` `.iconButton`。
- 不用 `:global`,除非包裹第三方库。
- 组合用 `composes`,不靠继承选择器。

## 模块差异变量

某模块需要跟全局不同的某值,在该模块根组件的 CSS Module 内声明:

```css
/* nav/nav.module.css */
.navRoot {
  --nav-bg: var(--color-surface-2);
  --nav-gap: var(--space-2);
  background: var(--nav-bg);
  gap: var(--nav-gap);
}
```

子组件用 `var(--nav-bg)` 读到模块值。不污染全局 `:root`。

## 消除硬编码

逐文件清理是渐进任务。新增/改动组件时,顺手把触碰到的硬编码值换成 token。不在本次大改里追全部历史文件。
