# 03 · 样式规范

## 三层分层

样式分三层，从上到下覆盖：

| 层 | 文件 | 内容 |
|---|---|---|
| 全局 token | `src/app/tokens.css` | spacing / text / leading / radius |
| 主题 | `src/app/theme.css` | 颜色、字体、阴影、过渡、焦点 |
| 模块覆盖 | 模块根组件的 CSS Module | 仅该模块自己的变量差异 |

`globals.css` 顶部固定顺序：

```css
@import './tokens.css';
@import './theme.css';
```

原则：

- token 先于 theme。
- token 不放颜色。
- theme 可以引用 token。
- 模块变量只在模块根容器里声明，不污染全局 `:root`。

## 主题语义

来自 `theme.css` 的常用语义变量包括：

- 文本：`--foreground` `--muted-foreground`
- 主色：`--primary` `--primary-foreground` `--primary-hover` `--primary-active`
- 语义色：`--success` `--warning` `--info` `--destructive`
- 背景：`--background` `--highlight` `--muted` `--secondary`
- 边框 / 输入：`--border` `--border-strong` `--input`
- 浮层：`--card` `--popover` `--overlay`
- 阴影 / 焦点 / 过渡：`--shadow-sm` `--shadow` `--shadow-lg` `--focus-outline` `--transition`

不要在 spec 里写不存在的变量名，例如 `--text-md`、`--text-primary`。当前正确写法分别是 `--text-base` 和 `--foreground`。

## 使用规则

推荐写法：

```css
.button {
    padding: var(--space-2) var(--space-4);
    font-size: var(--text-sm);
    line-height: var(--leading-normal);
    color: var(--foreground);
    background: var(--primary);
    border-radius: var(--radius-none);
}
```

禁止：

- `padding: 12px`
- `font-size: 14px`
- `color: #333`
- 在 `tokens.css` 里放颜色
- 为模块差异新建全局 `modules/` 样式层

## CSS Modules

- 一个组件配一个同名 `.module.css`
- class 名统一 camelCase
- 非必要不用 `:global`
- 组合优先用 `composes` 或 class 组合，不搞继承链花活

## 模块变量

模块需要局部变量时，只在模块根容器声明：

```css
.navRoot {
    --nav-bg: var(--secondary);
    --nav-gap: var(--space-2);
    background: var(--nav-bg);
    gap: var(--nav-gap);
}
```

子组件只读取 `var(--nav-bg)`，不要回写全局变量。

## 历史硬编码怎么处理

清理硬编码是渐进任务，不要求一次性全仓扫完。规则是：

- 新写组件必须直接用 token / theme 变量
- 改到旧组件时，顺手把碰到的硬编码替掉
- 不为了“纯洁度”做一轮只改样式值、不改功能的大面积重写
